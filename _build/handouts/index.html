Intro to JDBC

Introduction

The goal of this project is to create a text-based CLI program that reads user input and executes SQL commands to a Postgres/MySQL database using JDBC. This will allow you to familiarize yourself with the purpose of object relational mappers (ORM), libraries of code that allow you to interact with a database from traditional programming languages like Java/Python/etc. as opposed to pure SQL.

On the Support of IDEs

Throughout this exercise, we‚Äôll be running commands directly in a command line interface rather than utilizing built-in functionality within IntelliJ or another IDE. This is intentional‚Äì please follow the instructions exactly as you see them here. While there are alternative (perhaps slightly easier) ways of accomplishing these tasks within the IDE, we want you to get some experience running commands related to Maven, JDBC, and Java directly on the command line.

Think of it like you‚Äôre eating your cognitive vegetables for the day. ü•¶

But first‚Ä¶

Let‚Äôs review some ideas about the things you will use in this exercise so that you have some context for the different pieces of code you will be building today.

First, the Java Database Connectivity, or JDBC is part of Java‚Äôs standard library

  • JDBC allows a Java program to connect to a database, execute queries, and view results

  • The queries themselves are written in SQL, so you need to know that first

  • JDBC is ‚Äúbuilt-in‚Äù to the Java language, similar to how Collections (Array, Map, Set) are already present in Java

  • The packages java.sql and javax.sql are JDBC (javax.sql is JDBC v2.0)

  • You will see these being imported in your main file during this exercise

But what is JDBC, you may be wondering. JDBC is just a bunch of interfaces. As you know, interfaces are like blueprints for developers to implement - interfaces themselves don‚Äôt contain any business logic or actual code. Java leaves the implementation of the JDBC interfaces up to vendors of database systems (Postgres, MySQL, Oracle, etc.).

Database programs like PostgreSQL, MySQL, etc. are software applications entirely separate from your Java programs, not unlike Microsoft Word, Google Chrome, Spotify, Adobe Photoshop, etc. Even though they are separate from Java, they can be interacted with inside a running Java program with the help of JDBC.

JDBC ‚ÄúDrivers‚Äù are implementations of the interfaces for different databases. There is a driver for Postgres, a driver for MySQL, and so forth. While JDBC is part of the Java language, the drivers for each database are not part of the Java language. The creators of OracleDB wrote their own code to implement JDBC, as did the creators of MySQL, Postgres, MariaDB, H2, Apache Derby, etc.

The database driver is a dependency

Whichever database application you use (we‚Äôll be using Postgres), you‚Äôll need the corresponding JDBC Driver to be able to successfully connect to that database. So, in this exercise, you will use Maven to manage the installation and successful building of your project with the Postgres database driver dependency.

Add Driver Dependency

Next, download the code materials for this exercise on Frodo. Click here to download.

The bulk of the code you already have is a single file called JdbcIntro.java and pom.xml.

JdbcIntro.java contains some boilerplate code to get you started on JDBC. We‚Äôll revisit that in a moment. Now, we will be editing pom.xml, which is where we will store our project‚Äôs dependencies.

Add the Postgres driver dependency to your pom.xml file. Look for the <dependencies> section. There is a comment indicating where to add this in the given pom.xml.

<!-- https://mvnrepository.com/artifact/org.postgresql/postgresql -->
<dependency>
    <groupId>org.postgresql</groupId>
    <artifactId>postgresql</artifactId>
    <version>42.2.23</version>
</dependency>

Next, add the Maven Assembly plugin to your pom.xml file. Maven Assembly allows us to be able to package our Java code project, including the driver dependency, using Maven. Look for the <plugins> section. There is also a comment indicating where to add this in the given pom.xml.

<plugin>
  <artifactId>maven-assembly-plugin</artifactId>
  <executions>
    <execution>
      <phase>package</phase>
      <goals>
        <goal>single</goal>
      </goals>
    </execution>
  </executions>
  <configuration>
    <descriptorRefs>
      <descriptorRef>jar-with-dependencies</descriptorRef>
    </descriptorRefs>
  </configuration>
</plugin>

Step 5: JDBC Code Review

Open the file JdbcIntro.java and review the code. Here is a summary of what‚Äôs happening in the main method.

  • Get a Connection object using DataSource

  • Create a Statement object from the Connection object

  • Execute a query from the Statement, optionally storing the results in a ResultSet object

  • Loop through the ResultSet object to see the data

public class JdbcIntro {
    public static void main(String[] args) {

        // Using JDBC with Postgres Driver to query an already-existing database

        try {
            Connection connection = DriverManager.getConnection(
                    "jdbc:postgresql://localhost:5432/culinarydb",
                    "YOUR_POSTGRES_USERNAME",
                    "YOUR_POSTGRES_USER_PASSWORD");

            if (connection.isValid(500)) {
                System.out.println("Got a connection using DriverManager!");

                Statement statement = connection.createStatement();
                ResultSet resultSet = statement.executeQuery("SELECT * FROM foods");
                while (resultSet.next()) {
                    System.out.println(resultSet.getString(1) + " scores " + resultSet.getString(2) + "/10");
                }
            }

        } catch (Exception e) {
            System.out.println("Not able to connect & execute because: " + e.getMessage());
        }
    }
}
